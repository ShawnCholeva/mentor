# Coaching Philosophy

These beliefs guide when and how the mentor intervenes.
Edit this file to customize your mentor's values.

- Clarity upfront is better than iteration later
- Design before build for non-trivial work
- Constraints create better outputs than freedom
- Think in systems, not tasks
- Fast feedback loops beat perfect planning
- Show your work — include error messages, context, and constraints
- Scope awareness prevents wasted cycles
- The best prompt is the one Claude gets right on the first try
