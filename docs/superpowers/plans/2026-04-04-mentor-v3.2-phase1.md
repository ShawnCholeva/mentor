# Mentor v3.2 Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the broken coaching data pipeline, eliminate false positives on mid-conversation prompts, and enrich interaction logging so the user model can evolve on real data.

**Architecture:** Three surgical changes to existing bash hooks. The session state file becomes the data bridge between `user-prompt-submit` (writes coaching details) and `stop-logger` (reads and logs them). The evaluator gains conversation awareness by counting the history array it already receives. No new files, no new dependencies.

**Tech Stack:** Bash, jq, Claude Code hooks API

---

## File Map

| File | Role | Changes |
|---|---|---|
| `hooks/user-prompt-submit` | Main hook — evaluates prompts, writes coaching state | Write full coaching state (triggered, type, message) to session state file |
| `hooks/stop-logger` | Logging hook — reads state, appends interaction log | Remove `tags`; stop deleting state file; add `prompt_summary`; fix intent classification |
| `hooks/lib/evaluate-prompt.sh` | Evaluator — calls Claude to judge prompts | Derive turn count from history; add conversation-awareness rules |

- [ ] **Step 1: Compute TURN_COUNT from the history array**

- [ ] **Step 2: Add conversation-awareness section to the system prompt**

In `hooks/lib/evaluate-prompt.sh`, find the system prompt string. After the `## Rules` section (which ends around line 96 with the JSON response format), insert a new section before the closing single quote of `SYSTEM_PROMPT`.

Add this block after rule 9 (the last rule about pattern detection) and before the `Respond with ONLY a JSON object` line:

```bash
## Conversation Awareness
This is turn ${TURN_COUNT} of the conversation (${TURN_COUNT} previous prompts in session history).

If turn count > 1, the user is mid-conversation. Claude already has the full conversation context.
- References to prior discussion ("option 1", "the first one", "tell me more", "that approach", "both", "yes do X", "I'd like to see") are normal follow-ups. Do NOT flag these as lacking context.
- Short prompts (under 6 words) at turn > 1 are conversational, not vague. Only intervene if the prompt is truly ambiguous even WITH full conversation context.
- Only intervene on mid-conversation prompts for substantive issues: scope creep, missing diagnostics for debugging, fundamentally flawed approach. Missing context is never a valid reason to intervene after turn 1.
```

The specific edit: in the SYSTEM_PROMPT variable, find the line:

```
9. If you notice a pattern across the recent prompt history (repeated vagueness, improving specificity, etc.), weave that observation in naturally. Do not start with "Pattern:" or label it mechanically.
```

And insert the conversation awareness block after it, before:

```
Respond with ONLY a JSON object, no markdown, no explanation:
```
